SOP-001
Create a New Development Project
Use when: a new product idea has been approved for development and needs a controlled project record in the system.
  1. Click Projects in the left sidebar.
  2. Click New Project (top right, indigo button).
  3. Confirm or edit the Project Code. The format is PRJ-YYYY-### (e.g., PRJ-2026-003). Do not reuse codes from prior projects.
  4. Enter the Project Name (e.g., "Zen Silver Shot").
  5. Select the Category (e.g., Functional Beverage).
  6. Set Priority: Low / Medium / High / Critical. Use Critical only for projects with active commercial commitments or regulatory deadlines.
  7. Set Starting Gate to G0 for all new projects.
  8. Enter the Owner — the person driving the project forward.
  9. Enter the CMO if a manufacturing partner is already assigned (optional).
  10. Write the QTPP (Target Product Profile). Include: format, serving size, target actives with dose ranges, flavor/sensory direction, and shelf-life requirement.
    • Example: "Ready-to-drink 2 oz shot. Relaxation stack: Ashwagandha KSM-66 300mg + Chamomile 200mg. Citrus-forward flavor. 12 months ambient shelf life."
  11. Enter Target Actives — list each key ingredient and its target dose.
  12. Enter the Shelf-Life Target (e.g., "12 months ambient").
  13. Enter the Target Sensory Profile (e.g., "Light citrus, subtle floral, clean finish. No bitterness.").
  14. Click Create Project.
  15. Immediately record a G0 gate decision to formally document the intake triage. See SOP-004.
SOP-002
Enter a New Formula (First Version)
Use when: creating the very first formulation for a project.
  1. Click Projects → open the project you are formulating for.
  2. Click New Version (top right, indigo button).
  3. Set the Version Label to v0.1. All pre-freeze versions use the v0.x naming convention.
  4. Enter a descriptive Formulation Name (e.g., "Initial Draft", "Baseline Citrus Build").
  5. In What Changed, describe the starting rationale — what you are trying to achieve with this version.
  6. In Label Impact, note any ingredient choices that have immediate label implications. Leave blank for a first version.
  7. Add each ingredient in the Ingredients table. For each row, fill in:
    • Name (match the Materials master if the ingredient is registered there)
    • Supplier
    • Amount and Unit (e.g., 300 mg, 15 mL, 40 g)
    • Percentage (optional)
    • Category (Active, Flavor, Sweetener, Acidulant, Base, etc.)
  8. Write Process Notes — mixing sequence, temperatures, critical steps.
  9. Write Target Profile — what this specific version is meant to achieve sensorially.
  10. Confirm Created By is your name. Set Initial Status to Draft.
  11. Click Create Formulation.
ℹNext step is typically to produce a sample batch and collect feedback. See SOP-005 and SOP-007.
SOP-003
Revise an Existing Formula
Use when: feedback has been collected and changes need to be made, or a new iteration is required based on design decisions.
  1. Open the project → Formulations tab.
  2. Find the version you are branching from and click the Revise button (branch icon). The form opens pre-filled with all ingredients, process notes, and target profile from the parent version.
  3. The Version Label auto-increments (v0.2, v0.3, etc.). Adjust if needed.
  4. Update the Formulation Name to describe the change (e.g., "Reduced Citric + More Chamomile").
  5. Fill in What Changed — this is the most important field. Write a complete description of every change made and the reason for each one. Reference specific feedback scores or data that drove the decision.
    • Example: "Reduced citric acid from 2g to 1.5g — aftertaste scored 4/10; panel cited harshness. Bumped chamomile 200mg → 250mg — panel requested stronger floral note."
  6. Fill in Label Impact if any change requires a label update (e.g., new allergen source, sweetener type change, new active ingredient).
  7. Edit the ingredients table, process notes, and target profile as needed.
  8. Click Create Revision.
⚠Do not delete old formula versions. The full version history is the audit trail. All previous versions must remain visible in the timeline.
SOP-004
Record a Gate Decision
Use when: a gate review has occurred and the decision needs to be formally documented to advance, hold, or recycle the project.

Valid gate decisions

DecisionMeaningWhat happens in FormuLab
GOEvidence reviewed and accepted.Project stage advances to the next gate.
GO with ConditionsProject advances but specific conditions must be formally closed.Stage advances; condition details, owner, and due date are required.
HOLDBlocking issue found. Downstream work is suspended.Stage does not advance. A subsequent GO decision lifts the hold.
RECYCLEProject returns to an earlier stage.Stage does not auto-change; manually set the stage back after recording.
  1. Open the project → click Record Gate (violet button in the header). The gate pre-fills to the project's current stage.
  2. Confirm the correct Gate is selected in the dropdown.
  3. Select the Decision (GO / GO with Conditions / HOLD / RECYCLE).
  4. Enter the Actual Review Date.
  5. Enter the Planned Date if different from actual (optional, used to track schedule adherence).
  6. Enter all Reviewers present (comma-separated).
  7. Write Evidence Notes — summarize what was reviewed, what was confirmed, and any key findings. Be specific.
  8. If decision is GO with Conditions, fill in the condition panel that appears:
    • Condition Description — the specific deliverable required
    • Condition Owner — person responsible for closing it
    • Due Date
  9. Click Record Decision.
ℹA GO at G4 automatically sets the project's Formula Frozen flag. See also SOP-013 for the full G4 freeze procedure.
⚠Gate records cannot be edited after saving. To correct a mistake, delete the record (trash icon in the Gate Reviews tab) and re-enter it.
SOP-005
Log a New Sample Lot
Use when: a physical sample batch has been produced and needs to be entered into the sample control system (QP-245).

Sample classes

ClassWhen to useKey requirement
Class AInternal reference, non-ingestion (analytical standards, visual mocks, packaging samples)Minimal controls. No ingestion. Internal only.
Class BInternal ingestion (tasting by employees inside the org)Label issuance and second-person verification required before release.
Class CExternal distribution, or any sample where ingestion is possible outside controlled custodyStrictest controls. Allergen/ingredient disclosure. QA approval required before distribution.
  1. Click Samples in the sidebar → Log Sample. (Or go to Formulation Detail → Samples tab → Log Sample to link directly to a specific formula version.)
  2. Select the Project and Formula Version the sample was made from.
  3. Enter the Batch Code — your internal production reference (e.g., ZCB-003).
  4. Select the Sample Class using the table above as a guide.
  5. Select the Site where the sample was produced or will be stored.
  6. Enter the Date Produced.
  7. Enter Quantity and Unit (e.g., 30 bottles).
  8. Select the Storage Condition (Ambient, Refrigerated 2–8°C, Frozen, etc.).
  9. Enter the Expiry / Review Date — the date QA must review this lot's status.
  10. Enter Intended Use (e.g., "Internal sensory panel", "External lab analytical testing", "Consumer focus group").
  11. Check Ingestion Possible if anyone will taste or consume the sample.
  12. Check External Distribution if the sample will leave the building.
  13. Click Log Sample. The RSLN is auto-generated in the format RDS-YYMMDD-SITE-IN-SEQ and assigned to the record.
⚠If Ingestion Possible is checked and the class is not C, the app will display a warning. Per QP-245, ingestion-possible samples must be Class C.
SOP-006
Update a Sample's Lifecycle Status
Use when: a sample lot's physical state has changed — it has been released, shipped, consumed, destroyed, etc.
  1. Click Samples in the left sidebar.
  2. Find the sample using the search bar (search by RSLN, batch code, formulation, or project name).
  3. In the sample's row, use the lifecycle status dropdown on the right side. Changes save immediately.

SOP-007
Submit Sensory Feedback
Use when: a tasting or evaluation session has been completed and scores and notes need to be recorded against a specific formula and sample lot.
  1. Click Formulations → click the formulation version that was evaluated.
  2. Click the Feedback tab → Add Feedback.
  3. Enter the Reviewer Name.
  4. Select the reviewer's Role: Internal Tester / Consumer Panel / Customer / Scientist / Manager / External Reviewer.
  5. If more than one sample lot exists for this formulation, select the specific Sample batch that was evaluated.
  6. Use the sliders to rate each attribute 1–10:
    • Taste — overall flavor quality
    • Aroma — smell before and during consumption
    • Appearance — color, clarity, visual appeal
    • Texture — mouthfeel, viscosity, carbonation
    • Aftertaste — the finish after swallowing
    • Overall — holistic score
  7. In What worked well, record specific positives. Be precise — "honey sweetness is natural and rounds out the citrus" is more useful than "tastes good."
  8. In What didn't work, record specific issues (e.g., "aftertaste is harsh and metallic — possibly the citric acid level").
  9. In Suggestions for next iteration, record actionable changes the reviewer would want to see.
  10. Check Recommend reformulation before moving forward if another revision is needed before any approval or distribution decision.
  11. Click Submit Feedback.
✓The radar chart and average score ring on the formulation detail page update automatically as new feedback is submitted.
SOP-008
Add or Update a Material in the Materials Master
Use when: a new ingredient is being introduced, or a material's COA or qualification status has changed.

Adding a new material

  1. Click Materials in the left sidebar → Add Material.
  2. Enter the Material Name using the supplier's exact specification name for traceability (e.g., "Ashwagandha Extract KSM-66" not just "Ashwagandha").
  3. Enter the Supplier name.
  4. Select the Category (Active/Botanical, Flavor, Sweetener, Acidulant, etc.).
  5. Set the initial COA Status:
    • Pending — COA not yet received
    • Received — COA in hand, not yet reviewed
    • Approved — COA reviewed and accepted by QA
    • Rejected — COA reviewed and not acceptable
  6. In Notes, record specification details, allergen declarations, grade (food, USP), standardization level, and any qualification notes.
  7. Check R&D Approved if cleared for prototype development use.
  8. Check Commercial Ready if the supplier is qualified for commercial-scale production. A material can be R&D Approved but not yet Commercial Ready.
  9. Click Add Material. A code (MAT-####) is auto-assigned.

Updating an existing material

What you need to updateHow to do it

R&D ApprovedClick the R&D button on the row to toggle on (green) or off. Saves immediately.

SOP-009
Create Sample Labels and Shipping Documents
Use when: physical sample labels or a Chain of Custody document needs to be prepared before a sample lot is distributed or shipped.
ℹFormuLab does not yet generate printable labels or CoC documents. Label generation (LBL-QP-245-01/02/03/04) and Chain of Custody records (FRM-QP-245-06/07) are planned future platform modules. The procedure below bridges the current gap.

Step 1 — Get the RSLN from FormuLab

  1. Log the sample lot in FormuLab per SOP-005 before preparing any labels.
  2. Click Samples → find the lot. The RSLN is shown in indigo monospace text directly below the batch code (e.g., RDS-260322-ECS-IN-002).
  3. Note this RSLN — it must appear on all physical labels and shipping documents for this lot.

Step 2 — Prepare the physical label

Step 3 — Shipping and Chain of Custody

  1. Before shipping, confirm the lot status is Released in FormuLab.
  2. Prepare the Chain of Custody record (FRM-QP-245-06) using the current Word/Excel template. Reference the RSLN and batch code from FormuLab on the CoC document.
  3. Update the sample lifecycle status to In Transit in FormuLab per SOP-006 at the time of shipment.
  4. When the recipient confirms delivery, update the status to At Recipient.

SOP-011
View the Pipeline and Compliance Status
Use when: preparing for a management review, QA audit, or checking for overdue actions and open conditions.
  1. Click Dashboard in the left sidebar.
  2. The Stage-Gate Pipeline shows the count of active projects at each gate. Click any project name to open it directly.
  3. The top stats row shows Open Conditions, Projects on HOLD, Materials Pending COA, and Frozen Formulas.
  4. For condition details and due dates, click Gate Reviews in the sidebar. The Gate Pass Rate by Stage table shows pass rates per gate — a consistently low rate at a specific gate may indicate systemic evidence gaps.
  5. For materials readiness, click Materials → use the COA filter to show only Pending or Received materials. These need follow-up before a project can advance to G4 or G5.

SOP-013
Freeze a Formula at Gate 4
Use when: the G4 gate review has been passed with a GO decision and the selected candidate formula is being locked as V1.0.
⚠Formula freeze is a one-way, compliance-level action. Post-freeze formulation changes require formal change control documentation. Do not freeze a formula before the G4 gate review is complete.
  1. Confirm the project is currently at G4 stage (check the stage stepper on the project detail page).
  2. If not already done, record a G4 gate decision with decision = GO per SOP-004. This automatically sets the Formula Frozen flag on the project.
  3. Open the project → Formulations tab.
  4. Identify the candidate formula version to freeze (the version that passed the G4 review — typically the one currently In Testing or Approved).
  5. If the version label is not already v1.0, note this — ideally the label is updated to v1.0 before freezing. You can create a revision labeled v1.0 and immediately freeze that revision.
  6. Click the Freeze button (lock icon, violet) on the formula card.
    • The Freeze button only appears when the project stage is G4.
  7. The formulation card turns violet with a lock icon and "Frozen" label. Status is set to Approved.
  8. The project header now shows a Formula Frozen pill.
ℹAny new formulation version created after freeze will display a warning: "Post-freeze changes should go through formal change control." Label post-freeze versions as v1.1, v1.2, etc., and document the change control rationale in the What Changed field.
INFO
Data Storage and Backup

All FormuLab data is stored in your browser's local storage on the device you use. Data persists between sessions but is not backed up to a server or shared automatically between users.

⚠Do not clear your browser's site data or cache for the FormuLab URL — this will permanently delete all project records with no recovery path. Periodically export records or take screenshots of critical data as a backup measure.

Multi-user shared access and server-based storage are planned future enhancements. For now, designate one machine as the primary FormuLab workstation and ensure team members enter records on that device.
